### Issue #3: Toast Context Subscription Causes Unnecessary Re-Renders ‚ö†Ô∏è

**Location:** `SubmissionDetail.jsx` (Line 254)

**Problem:**

**How Toast Context Works:**
1. Every toast call (`toast.loading()`, `toast.success()`, `toast.error()`) updates `toasts` state in ToastProvider

**Impact:**
- Download click ‚Üí `toast.loading()` ‚Üí SubmissionDetail re-renders
- Toast success ‚Üí `toast.success()` ‚Üí SubmissionDetail re-renders again
- 2 unnecessary re-renders per download!
